One-Way Binding

Property Binding [ ]

Square brackets tell Angular to evaluate the right-hand side as a TypeScript expression and assign the result to the named DOM property — not an HTML attribute.

Syntax: [property]="expression"

property is the DOM property to bind to, and expression is the TypeScript value to assign.

Key difference from attribute binding: you're setting JavaScript properties (e.g. img.src), not HTML attributes (e.g. src="..."). Property binding is used to set element state — like enabling/disabling a button based on component logic.

Disabled Property

Binds to the [disabled] DOM property. Without brackets, any non-empty string would always disable the button — because HTML attribute strings are truthy.

<button [disabled]="isButtonDisabled" (click)="onButtonClick()">

Image src, alt & width

Binds to [src], [alt], and [width] DOM properties of the <img> element using component data.

<img [src]="imageSource" [alt]="imageAlt" [width]="imageWidth" />

Anchor href Property

Binds [href] to a dynamic URL from the component. Interpolation {{ linkUrl }} displays the URL as visible text.

<a [href]="linkUrl">{{ linkUrl }}</a>

Input Placeholder

Binds to the [placeholder] DOM property of an input element, pulling the hint text dynamically from the component class.
